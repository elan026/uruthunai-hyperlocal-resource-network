# 🚨 Emergency Mode (Override)

**Emergency Mode** is a "Deep Resilience" feature that shifts the entire platform into a hyper-focused state during massive localized disasters.

### Key Features
- **UI Simplification**: When active, the general Dashboard transforms into an "Emergency Dashboard."
- **Critical Focus**: Low-priority resources are hidden or de-emphasized. High-priority items (Medical, Shelter, Water) are moved to the top.
- **Evacuation Tracking**: Highlights nearby Shelter Capacity and Evacuation statuses.
- **Global Override**: A master switch controlled by lead Admins that changes the theme and UX experience for all users in the affected zone.
- **Power Savings**: Uses a more high-contrast, simpler visual structure intended to be readable in low-light/storm conditions.

### Workflow
1. **Activation**: A lead admin toggles the "System Red" state in the Emergency Control Center.
2. **Transition**: Every user’s site header changes colors and shows an active "System Emergency" status.
3. **Execution**: The main dashboard priority shifts to "Live Critical Requests" and shelter counts.
4. **Shutdown**: Once normalized, the admin reverts the state, returning the platform to standard resource-sharing mode.
