# 📢 Hyperlocal Alert System

The **Hyperlocal Alert System** is a platform-wide communication engine for broadcasting verified, high-urgency information.

### Key Features
- **Multi-Level Urgency**:
    - 🔵 **Info**: General community updates (e.g., Water tanker arrival).
    - 🟠 **Warning**: Potential hazards (e.g., Heavy rain forecast).
    - 🔴 **Critical**: Immediate danger (e.g., Evacuation order or Reservoir discharge).
    - 🟢 **Resolved**: All-clear signal.
- **Dynamic Timeline**: A chronological feed of all alerts posted by local administrators.
- **Admin-Only Posting**: To prevent rumors, only verified admins can generate "Global Alerts."
- **Landing Page Integration**: The most critical "Red Alerts" are mirrored directly on the homepage banner for maximum visibility.

### Workflow
1. **Observation**: Local authorities or admins identify a critical occurrence.
2. **Broadcast**: Admin selects an urgency level, writes an actionable message, and publishes.
3. **Reception**: The alert appears instantly on every user's "Community Alerts" feed.
4. **Resolution**: Once the event passes, the admin marks the alert as "Resolved" to declutter the feed.
